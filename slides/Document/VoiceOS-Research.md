# Voice OS エージェントモード 調査結果

## 結論
Voice OSのエージェントモードは、音声だけでSlack、Gmail、カレンダーなどを自動操作できる機能です。3時間のハッカソンでは、音声コマンドから複数のアクションを実行する「ワークフロー定義」を作成します。

---

## 1. エージェントモードとは？

Voice OSには3つのモードがあります：

| モード | 説明 |
|--------|------|
| **通常モード** | 「あー」「えー」などの無駄な言葉を自動削除 |
| **指示モード** | 話した内容をプロンプト化してLLMに送信 |
| **エージェントモード** 🆕 | Slack、Gmailなどを音声で直接操作可能 |

エージェントモードの特徴：
- ✅ リアルタイムで音声対話
- ✅ 複数のタスクを自動で実行
- ✅ アプリを実際に操作（情報取得だけでなく）

## 2. 具体的にできること

### A. Slack操作の例
```
あなた: 「チャンネル #general に『プロジェクト完了』と送って」
↓
エージェント: Slackに自動でメッセージ送信
```

### B. Gmail操作の例
```
あなた: 「田中さんに明日10時の会議のリマインドを送って」
↓
エージェント: 
  1. 田中さんのメールアドレスを検索
  2. メール本文を自動作成
  3. 送信
```

### C. カレンダー操作の例
```
あなた: 「今月の空いている時間を教えて」
↓
エージェント:
  1. Googleカレンダーを読み込み
  2. 予定を分析
  3. 空き時間を音声で報告
```

### D. 複合ワークフローの例
```
あなた: 「営業報告書を作って、田中さんに送って、Slackで通知して」
↓
エージェント:
  1. Google Docsで報告書作成
  2. メールで田中さんに送信
  3. Slackの#営業チャネルで通知
  4. 各ステップの進捗を音声報告
```

## 3. ワークフロー定義の方法

### パターン1: シンプルなコマンド（初心者向け）
```yaml
command: send_slack_message
trigger: "voice"
parameters:
  channel: "#general"
  message: "プロジェクト完了"
```

### パターン2: 複雑なワークフロー（上級者向け）
```typescript
const workflow = {
  name: "Daily Report",
  steps: [
    { type: "voice_input", action: "listen" },
    { type: "fetch_calendar", source: "google_calendar" },
    { type: "fetch_emails", source: "gmail" },
    { type: "generate_summary", model: "claude-3" },
    { type: "voice_response" }
  ]
}
```

### パターン3: 二つのエージェントを使う方法
```
Voice Agent (会話担当)  ←→  Action Agent (実行担当)
      ↓                          ↓
  「Slackを開いて」         Slack APIを呼び出し
  「メッセージ送って」       実際に送信
```

## 4. 音声インターフェースの特徴と制約

### ✅ メリット
- **ハンズフリー**: キーボード不要で移動中も使える
- **自然**: 専門知識不要で誰でも使える
- **速い**: 話すだけなのでタイピングより早い
- **文脈保持**: 「さっきのメール、田中さんに転送して」が可能

### ⚠️ 注意点
| 課題 | 対策 |
|------|------|
| **固有名詞の間違い** | 確認質問を入れる（「太郎さんですね？」） |
| **時間がかかる処理** | 「実行中です」と即座に応答 |
| **音声の曖昧性** | 選択肢を提示（「返信を読む？作成？」） |
| **プライバシー** | 重要な操作は確認ステップを入れる |

## 5. ハッカソンで作るもの

### 目標
**「Voice OSのエージェントモードを活用したワークフローを定義したコマンド」**

### 制約条件
- ⏱️ **開発時間**: 3時間
- 🎤 **開発方法**: 音声のみ（タイピングなし）
- 🤝 **ペアコーディング**: チームで協力
- 💻 **ツール**: Claude Code（1万円分クレジット）

### 実装すべき要素
1. **音声入力** → ユーザーの指示を受け取る
2. **エージェント処理** → 指示を理解して必要なアクションを決定
3. **アクション実行** → Slack/Gmail/Calendar等を操作
4. **音声フィードバック** → 結果をユーザーに報告
