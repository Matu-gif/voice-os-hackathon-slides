# Voice OS ハッカソン スライドプロジェクト - ガイドライン

## プロジェクトまとめ

このプロジェクトは、**Voice OSハッカソン用のスライドを作成・管理する**ためのものです。

Voice OSとは音声でコンピュータを操作するツールで、そのハッカソンイベント（4/3開催）の準備として、イベント内容や今後やるべきことをまとめたスライドを作成しています。

---

## ファイル構成

プロジェクトは以下のように整理されています：

```
hackathon/
├── instructions.md           # このファイル（プロジェクト全体のガイドライン）
└── slides/
    ├── index.html           # スライド本体（ブラウザで開くHTMLファイル）
    └── Document/            # ドキュメント保管場所
        ├── Design.md        # スライドのデザイン指示（カラー、スタイル）
        ├── structure.md     # スライドの構成とイベント情報
        └── 決定事項.md      # 以前作成した要件定義ドキュメント
```

### 各ファイルの役割

- **`slides/Document/`**: ドキュメント類をまとめる場所
- **`Document/Design.md`**: スライドのデザインに関する指示（色の配分、スタイルなど）
- **`Document/structure.md`**: スライドの構成やハッカソンのイベント情報
- **`Document/決定事項.md`**: スライド作成時に決めた技術仕様やデザイン仕様
- **`slides/index.html`**: 実際に表示するスライド（HTML形式）

## 作業ルール

このプロジェクトで作業する際は、以下のルールに従ってください：

### 1. **結論ファースト**
- 説明や提案をする時は、まず結論を伝えてから詳細を説明する
- 例: 「〇〇を作成します。理由は△△だからです」

### 2. **日本語で分かりやすく**
- planや説明文は日本語で記述する
- 技術者ではない普通の人でも理解できる言葉で説明する
- 専門用語を使う場合は、簡単な説明を添える

### 3. **適切なDocumentの編集・作成**
- 作業内容に応じて、適切なドキュメントを編集または新規作成する
- **例**:
  - スライドのデザインに関することなら → `Design.md`を編集
  - イベント情報や構成に関することなら → `structure.md`を編集
  - 新しい決定事項があれば → `決定事項.md`に追記または新規ファイル作成

## 技術情報（参考）

### デザイン仕様
- **カラーパレット**: 白ベース（7割）、青（2.5割）、オレンジ（0.5割）
- **スタイル**: シンプルで信頼感のあるデザイン
- **技術**: HTML/CSS（単一ファイル、16:9のワイドスクリーン向け）

### スライド構成（全6枚）
1. タイトル
2. ハッカソンイベント情報
3. ハッカソン内容
4. 提供されるもの
5. Voice OSとは？
6. 今後やるべきこと
